Stage 0: 1.1 Learning the basics of HTML

0.1 What is HTML for?

HTML is used for web development. Almost all web pages uses HTML as its markup language. HTML means HyperText Markup Language. HTML is the most popular language used in web development.
And the good news is that HTML is relatively easy to learn.

0.2 HTML tags

HTML tags are what we use to tell a web browser how to display content in a web page.
Most HTML tags consist of an opening and a closing tag.

0.3 Text markdown tips

for bold you must use b to open a tag and /b to close it.
for italic you must use em and /em.

0.4 Examples

Hello, my name is Bold and I am a tool for people put their phrases in a strong way.
Hi, Mr Bold. I am italic, i am from Italy

0.5 Additional HTML tags:


p and /p defines a paragraph.
br creates a line break.
div and /div creates a "box" containter element
span and /span creates an "inline" container element.

0.6 HTML elements

HTML elements usually consists of a start tag and end tag, with the content inserted in between.

0.7 Inline and block - HTML elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them, stretching out to the left and right as far as possible. Block elements always start a new line while Inline elements does not. Inline elements only uses as much width as necessary.

Stage 1: 1.2 Creating a Structured Document

1.1 Developer tools

Access the Developer tools in Google Chrome using Crtl+Shift+I. So you can view the structure of a web page. The panel on the right is the styles panel, which is used for CSS. All visible elements are rectangular.

1.2 Tags and tree-like structures

Tags and ident form Tree Like Structures or DOM The browser turns HTML tags into elements that turn into a tree. The three-like structure refers to the fact that HTML elements can have other elements embedded in them.

1.3 DOM

Dom is the way the browser reads your code and turn it on a webpage. It is linked to the Tree-like Structure

1.4 CSS what is it?

CSS are a web development language that controls how the webpage will look. While HTML controls the structure, CSS controls the colors, fonts and style and sizes.

1.5 How it works?

Web developers have to give similar elements the same classname so then specify the style to be applied to that class.

1.6 The web page world is based on boxes

In webpages, all elements are in a box. They are all in boxes because it makes it easier to rearrange the website. So Web pages are simply boxes within boxes. Look at a web design as boxes, starting large and then smaller as this is how you'll write the HTML.

1.7 Boxes to HTML

Start by breaking down the boxes into div tags. Add content between tags to remember the content, eg. Header, body, footer. Add class names, be as specific as possible, think labeling moving boxes.

1.8 The Box Model

The box model includes the margin, border, padding and content. By default, the size of the actual element is the sum of the border, padding and content width.

1.9 The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

1.10 Span and Div

Span and div can both contain text. Span is an inline, and div is block. Once you have used Span or Div, you can attach CSS styles to change the way they look.

1.11 Text Editors (for coding)
